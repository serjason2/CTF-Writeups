# Kusto Query Language (KQL) 201 - Cheat Sheet
- I learned how to search and filter data in KQL 101. Now it's time to go deeper.
- In KQL 201, I'll learn advanced techniques that real security analysts use in a day-to-day basis:
  - Filtering events across time ranges
  - Aggregating thousands of logs into actionable insights
  - Transforming raw data into clear results

These skills will help me move faster, spot patterns that others could miss, and answer complex questions with a single query.

# What I'll Learn
- Filter events by date ranges using `datetime()`, `between`, and `ago()`
- Group events into time buckets with `bin()` to spot spikes in activity
- Use summarize with `count()`, `dcount()`, `min()`, `max()`, and `make_set()` to analyze patterns
- Shape query output with project and add calculated fields with extend
- Apply conditional logic using `iff()` and `case()`
- Search efficiently with `startswith`, `endswith`, `in`, and `has_any`
- Detect password spray attacks by analyzing failed login attempts
