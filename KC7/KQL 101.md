# Kusto Query Language (KQL) 101 - Cheat Sheet
**Syntax**
```
Table
| where <field> <operator> <value>
<more kql here if needed>
```

Example 1: Query below is only going to output 10 records if applicable (shows you column & values that are in the table if forgotten)
```
Employees
| take 10
```

Example 2: Query above is counting the amount of employees that are working at the company (based on table specified)
```
Employees
| count
```

---
**KQL Operators**
- **take**: to show columns & values that are in the specific table if forgotten
- **where**: to find a specific value
  - **contains**: will always return TRUE if the searched string exists within the searched text (basically any word that has "real" in it)
    - Example: contains "real"
      - realist, realme, real, dyarrealestate, dubairealestatedeals
  - **has**: result depends on the surrounding of the searched string within the searched text (only outputs the word “real”)
    - Example: has "real"
      - real.vision
- **count**: shows the amount of values specified in the table
- **distinct**: to find unique values in a specific column
- **let**: is basically creating a variable name and you can call it whenever you want instead of constantly rewriting the same query
  - Example:
```
  let mary_ips =
  Employees
  | where name has “Mary”
  | distinct ip_addr
  OutboundNetworkEvents
  | where ip_addr in (mary_ips)
  | distinct url
```
